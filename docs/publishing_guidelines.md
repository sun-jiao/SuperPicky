# 慧眼选鸟 (SuperPicky) 社交媒体与平台发布指南

为了确保在不同平台上发布内容时，排版、样式和格式能够完美适配对应平台的限制与特性，特制定此格式要求指南。AI 助手在撰写发布稿件时，**必须**参考本文档以采用与之匹配的排版方案。

## 1. YouTube 社区动态 (Community Post) & Bilibili 动态

这两个渠道主要面向视频粉丝的日常互动与版本通知，主要在移动端被阅读。

*   **技术限制：绝对的纯文本**
    *   **不支持**：任何 Markdown 语法（如 `**加粗**`、`*斜体*`、`# 标题`、`[超链接文本](url)`）均无效，会被作为明文显示，极为难看。
    *   **不支持**：任何 HTML 标签及 CSS。
*   **链接规范**：
    *   只能直接粘贴长链接（如 `https://github.com/jamesphotography/...`），平台会自动将其识别为可点击的超链接。
*   **排版与视觉要求**：
    *   **大量留白**：坚决做到**段落级空行**。不要将每一句话换行，而是保持完整的逻辑段落，在不同段落、不同功能块（如“亮点1”和“亮点2”）之间，**必须留出一个完全空白的空行**，以减轻手机端阅读的视觉压迫感。
    *   **视觉锚点**：大量且合理地使用 Emoji（如 🚀, 🔥, 🎯, ✨, 💡, 👇, ➡️）来代替普通的项目符号或强调作用。
    *   **列表替代**：使用 `•` 或数字 Emoji (`1️⃣`) 替代 Markdown 的 `-` 列表。

## 2. GitHub (README, Release Notes)

面向开发者、技术型摄影师的核心产品阵地。

*   **技术支持：GitHub Flavored Markdown (GFM)**
    *   **完全支持**：所有的标准 Markdown 语法（粗体、斜体、列表、表格、代码块 ` ``` `、行内代码 ` ` `）。
    *   **部分支持**：基础安全的 HTML 标签（如 `<details>` 与 `<summary>` 用于折叠长文本，`<img width="x">` 用于控制图片大小）。
*   **排版与视觉要求**：
    *   结构严谨，逻辑层级清晰。多用 `##` 和 `###` 构建大纲。
    *   使用原生的超链接语法 `[显示文本](URL)`。
    *   保持克制，偶尔可使用 Emoji 点缀，但不宜过多，以专业、清晰、直接为最高优先级。

## 3. 微信公众号 (WeChat Official Account)

面向中国大陆的成熟摄影师与核心私域流量，要求极高的画廊级排版美学。

*   **技术限制：严格的内联 HTML/CSS**
    *   **不支持**：Markdown 原文（必须在推文前转换为 HTML）。
    *   **不支持**：外部 CSS 文件、`<style>` 标签（经常被微信过滤）、JavaScript。
    *   **强制要求**：所有的样式必须写在标签的 `style="..."` 属性中（Inline CSS）。
*   **排版与视觉要求 (基于之前的微信推文标准)**：
    *   **深色/质感背景**：通常采用深灰/黑背景色（如 `#1a1a1a` 或 `#111`）。
    *   **图片样式**：图片必须具有画廊质感，使用圆角 (`border-radius: 8px`) 以及柔和的阴影 (`box-shadow: 0 4px 15px rgba(0,0,0,0.5)`)。为了手机端适配，宽度通常为 `width: 100%` 或 `max-width: 100%`。
    *   **字体排版**：采用优雅的无衬线字体（如 `system-ui, -apple-system, sans-serif`），行高宽裕 (`line-height: 1.8`)，字间距舒适 (`letter-spacing: 0.5px`)。
    *   强调文字常常使用非典型的颜色（如柔和的金色、突出的警示红）或调整透明度 (`opacity: 0.8`)。

## 4. 小红书 (Xiaohongshu)

面向泛摄影爱好者、年轻一代，追求情绪价值和瞬间吸引力的“种草”平台。

*   **技术限制：加强版纯文本**
    *   **不支持**：任何 Markdown 书写、HTML 和外链。
    *   **封杀外链**：在正文中**绝对不能**直接放网址链接或者诱导关注的链接（会被系统限流或屏蔽）。遇到需要展示网址的情况，只能写结论（例如：“去我的 GitHub 主页下载” 或 “链接见置顶评论”）。
*   **排版与视觉要求**：
    *   **极短的句子**：一句话不要太长。
    *   **海量 Emoji**：每一段甚至每一句的开头或结尾都带有强情绪的 Emoji。
    *   **吸睛标题**：第一句话必须足够惊悚/吸引人（例如：“家人们谁懂啊”、“摄影师选片神器来了”）。
    *   大量利用空行和 `#话题标签`（如 `#摄影日常`, `#鸟类摄影`, `#Lightroom插件`）放在文末。

---

## 5. Hashnode / 个人技术博客

技术深度分享区。

*   **技术支持：原生兼容的 Markdown**
    *   **支持**：常规 Markdown 语法。支持插入丰富的代码片段和嵌入框。
*   **排版与视觉要求**：
    *   可作为类似 GitHub 的长文输出地，语气可以更倾向于“经验分享”、“技术布道”或“架构解析”。
